# Contributing to CLAUDE.md

Thank you for your interest in contributing! This document provides guidelines for contributing to this project.

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs or suggest enhancements
- Search existing issues before creating a new one
- Provide clear, detailed descriptions with examples where applicable

### Submitting Changes

1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**
   - Keep changes focused and minimal
   - Follow existing style and conventions
   - Test your changes with Claude Code or Claude Desktop
4. **Commit your changes**
   - Write clear commit messages describing why, not what
   - Use present tense ("Add feature" not "Added feature")
5. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```
6. **Open a Pull Request**
   - Describe what changed and why
   - Reference any related issues

## Code Style

- Use markdown for all documentation
- Keep CLAUDE.md instructions clear and actionable
- Organize by category (Execution, Decision-Making, Communication, etc.)
- Use bullet points for guidelines
- Bold key phrases for scannability

## Testing

Before submitting:
- Verify markdown syntax is valid
- Test instructions with Claude Code or Claude Desktop
- Ensure changes don't conflict with existing preferences

## What to Contribute

Useful contributions include:
- Bug fixes in documentation
- Clarifications to existing instructions
- New preference categories that are broadly applicable
- Examples demonstrating preference usage
- Improvements to README or supporting documentation

## What Not to Contribute

Please avoid:
- Highly project-specific preferences (these belong in project-level CLAUDE.md files)
- Contradictory or conflicting instructions
- Overly verbose or marketing-style language
- Features that require external dependencies

## Questions?

Open an issue for discussion before making large changes.
